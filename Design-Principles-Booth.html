Complexity and Simplicity

  • A site should only be as complex as it needs to be
  • Every site has a mandatory level of complexity
  • A user should gain functional experience with a site very fast
  • Users with functional site experience should not need to think when using the site

Inclusiveness

  • Always ask what disabilities does this site not support
  • Text-to-speech is always a great option
  • Having alternate color and text schemes for the visually impaired is important
  • Provide haptic/sensory feedback for supported devices

Cultural

  • Consider Societal ethics (i.e. individualism vs communitarianism)
  • Remember some cultures read left to right
  • Research potential metaphors before using them (ex. In some countries a thumbs up is a sign of disrespect, so you should avoid using it as a positive metaphor)
  • Avoid using United States standards for sites that may have worldwide reach, like having prices only is USD or putting the US at the top of a country drop-down list

Ease of Use

  • Use metaphors for interactions
  • Take from nature/natural understanding when designing
  • Guide the user to actions they would want to take
  • Make sure users don't reactionarily make important decisions
  • All actions should have feedback
  • The website should use colors, feedback, and standard conventions to create affordance (i.e. the user should know what is interactable)
  • Follow Jakob's law - your site should look like other sites
  • Have helpful notifications or tips for complicated actions

Attention/Distraction

  • Only push notifications that are important
  • Understand how exterior distractions may affect your users experience
  • Consider people with ADHD and how distractions could affect them
  • Find the write times to give people helpful information and tips to avoid distractions

Other

  • Understand the users you are designing for
  • Anthropomorphism can be used to develop an emotional attatchment or positive reaction
  • AI can be useful, but only use AI for decisions if you can understand and back up the conclusion
  • Your site should have a consistent mood
  • Listen to user feedback and adjust your site as needed
  • Never trick the user into performing an action (follow ethical standards!)
